# Storyboard Color Grading Control / 数据与 HEX 调色控制

核心信念:色彩不是“氛围词”,而是可控的数据结构。不要让 AI 猜“复古”“电影感”“青橙色调”,要用 HEX 色卡、色彩 DNA、光比、色温和验证闭环接管画面的色彩。

这个 skill 提炼自两篇中文教程:
- 《如何不用提示词给画面调色? - HEX调色法》: <https://www.super-i.cn/info-2680.html>
- 《告别“AI塑料感”——数据驱动型电影级调色实战指南》: <https://www.super-i.cn/info-2611.html>

## 两条调色路径

### 路径 A:HEX 色卡法

适合账号视觉统一、系列图、品牌图、同风格视频首帧。

流程:

1. 收集 5-10 张同风格参考图。
2. 提取核心 HEX 色值,生成专属色卡。
3. 把色卡作为视觉参考喂给生图工具。
4. 静态图确定后,作为视频首帧。
5. 视频提示词只写动作,不要重复光影调色词。

提示:

```text
使用参考色卡锁定画面色域。主色控制在 #CBBFA2 / #8B7355 / #2F2A24 附近,避免高饱和红绿蓝;整体低饱和、暖灰、哑光胶片质感。
```

### 路径 B:数据驱动电影调色

适合复刻电影色调、去除 AI 塑料感、做专业风格迁移。

把参考图拆成:

- **Hue Distribution**:主要色相分布
- **Luminance Ratios**:明暗比例/光比
- **Color Temperature Bias**:色温倾向
- **Saturation Mapping**:饱和度映射
- **Shadow Tint**:暗部偏色
- **Highlight Roll-off**:高光过渡

## 色彩 DNA 提取指令

```text
请不要描述画面内容。请把这张参考图拆解成可复用的色彩 DNA:
1. 主导 HEX 色值 5-8 个,标注占比
2. 明暗比例与光比
3. 色温倾向
4. 暗部偏色与高光颜色
5. 饱和度策略
6. 材质与胶片颗粒
7. 负向约束:哪些颜色/质感必须避免
输出一段可直接嵌入 AI 生图提示词的 DIT 调色指令。
```

## 参数嵌入法

弱写法:

```text
电影感,复古蓝绿色调,暗一点,高级质感。
```

强写法:

```text
调色严格采用低饱和单色倾向。主导色为沙尘米黄 #CBBFA2、哑光棕 #8B7355、深炭灰 #1F1E1A。暗部注入极微弱青绿色偏色,高光为柔和奶油色,无纯白死亮。整体模拟 bleach bypass 跳漂胶片,高反差但保留暗部纹理,哑光表面,粗颗粒胶片质感。
```

## 图生视频色彩延续

首帧已经锁定色彩后,视频 prompt 只写运动:

```text
镜头缓慢向前推进,人物轻微转头,衣角被风带起。保持首帧的色彩体系、光影和胶片质感不变。
```

不要再写:

- cinematic color grading
- teal and orange
- high quality color
- movie style

## AI 视觉闭环验证

生成后不要只靠肉眼。把参考图和生成图交给视觉模型:

```text
请作为严格的 DIT 调色监督,对比图A生成图与图B参考图。只评价色彩和光影,不要评价构图和主体。
输出:
1. 色相偏差
2. 曝光偏差(EV)
3. 暗部/高光偏差
4. 饱和度偏差
5. 需要加入或删除的 HEX/调色指令
6. 匹配度百分比
```

## 出稿前检查

- 是否把“风格词”降维成 HEX、光比、色温、饱和度?
- 是否有负向色彩约束,如禁止高饱和红绿/纯白/HDR/油亮塑料?
- 系列图是否使用同一色卡?
- 视频是否用首帧延续调色,而不是重新写风格词?
- 是否经过 A/B 色彩验证和迭代?
