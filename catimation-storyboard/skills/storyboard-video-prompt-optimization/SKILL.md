# Storyboard Video Prompt Optimization / 视频提示词优化

核心信念:AI 视频提示词不是越长越好。图生视频里,首帧拥有视觉霸权;文本应该主要负责**驱动运动**。如果同时塞入风格词、冲突运镜、过多动作,模型会稀释运动权重,最终生成平庸画面。

这个 skill 提炼自中文教程《告别无效指令:AI视频提示词优化的三大核心法则》。参考: <https://www.super-i.cn/info-2794.html>

## 法则一:尊重首帧霸权

图生视频中,第一帧已经决定:

- 视觉风格
- 光影结构
- 人物长相
- 服装材质
- 色彩基调
- 基础构图

因此视频 prompt 不要重复:

- 超写实
- 电影级光影
- 8K
- 高级质感
- UE5
- 赛博朋克(如果首帧不是)
- 复杂风格标签

弱写法:

```text
赛博朋克光影,UE5顶级3D渲染,极度清晰,女孩缓缓转动一圈。
```

强写法:

```text
女孩在雪地中缓缓转过身,目光自然落向镜头;转身瞬间,寒风轻轻扬起她的发丝和深色衣角。
```

把文本权重留给动作、眼神、物理反馈。

## 法则二:数值要转成视觉参照

AI 不擅长把“风力6级、移动5米、每秒1米”换算成当前画面的像素尺度。改成可见结果:

弱写法:

```text
风力6级,女孩以每秒1米向前走5米。
```

强写法:

```text
强风把她的衣角明显吹向右侧,她只向前迈出一步,靴底在雪地里压出清晰脚印;她微微低头,呼出的白雾从嘴边散开。
```

## 法则三:确立单一主导逻辑

提示词内部不能互相打架。先决定镜头主导逻辑:

- **情绪主导**:固定机位/中近景/慢动作/微表情/局部小动作。
- **广告感主导**:平滑推轨/产品中心/光影稳定/高完成度动作。
- **纪录感主导**:轻微手持/自然跟拍/不完美遮挡。
- **动作主导**:明确一个核心运动,其他元素让位。

冲突例子:

```text
强烈肩扛手持镜头展现情绪张力,同时镜头极其平滑地向前推进。
```

问题:手持晃动和平滑推轨互相抵消。

正确做法:

```text
固定机位,维持原有柔和自然光;女孩静静低头,右手翻开书的下一页,纸张边缘轻轻弯曲,眼神跟随文字移动。
```

## 法则四:用时间线分配运动算力

视频 AI 的本质是运动算力分配。不要“一锅炖”:

弱写法:

```text
狂风刮过,信全部飞向天空,女孩猛地站起来追赶,无人机360度环绕。
```

强写法:

```text
0-2s: 固定镜头。风先掀动地上最前方几封信,纸角轻轻翻起,女孩仍蹲在原地。
2-5s: 风力增强,信纸从前景向中景依次被卷起,女孩抬头,手掌按住膝盖准备起身。
5-8s: 女孩缓慢站起向前迈一步,只追向最近一封信;镜头保持稳定,把运动权重留给信纸和人物起身。
```

每个时间段只给一个核心目标:

- 0-2s 给环境微动
- 2-5s 给主要物体运动
- 5-8s 给人物反应

## 视频 Prompt 清理流程

写完后做减法:

1. 删掉和首帧重复的风格/画质/光影词。
2. 删掉互相冲突的运镜和光影。
3. 只保留一个主运动。
4. 如果动作多,拆成时间线。
5. 把抽象数值改为画面可见参照。
6. 明确镜头是否固定,不要默认乱动。

## 出稿前检查

- 首帧已经决定的视觉信息,是否被重复写进视频 prompt?
- 文本是否主要描述运动、眼神、物理反馈?
- 是否存在手持 vs 平滑、强光 vs 原图柔光等冲突?
- 是否同时要求太多主体、环境和镜头动作?
- 是否按时间线分配每秒主角?
- 是否用可见参照替代抽象数值?
