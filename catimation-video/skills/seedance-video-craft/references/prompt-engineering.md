# Seedance 2.0 提示词工程

针对 Seedance 2.0 的真实特性写提示词。核心:**单一主导动作 + 首帧霸权 + 运动矢量语言 + 主动写声音 + 默认只写正向提示词(不写反向)+ 用真实技法词填充**。不要把它当成「写得越多越好」的图像模型。

---

## 法则 1:尊重首帧霸权(图生视频)

图生视频里,start_image 已经决定了:风格、光影、人物长相、服装材质、色彩基调、基础构图。

**所以提示词不要复述静态帧**,不要再堆:超写实 / 电影级光影 / 8K / UE5 / 赛博朋克(如果首帧不是)。

- ❌「一个棕发男人穿皮夹克拿咖啡,电影级光影,8K」
- ✅「他缓缓抬头,镜头慢推,咖啡热气上升」

文本主要负责**驱动运动**,不是重描画面。

---

## 法则 2:单一主导动作(别稀释运动权重)

4–15 秒里塞太多动作,模型会把运动算力摊薄,结果每个动作都做不到位 → PPT 感、动作敷衍。

- 一条片 = 一个清晰的核心动作 +(可选)一个次要反应。
- 复杂叙事 → 拆成多段,每段一个主导动作,再用 ffmpeg 拼接。

- ❌「她走进房间、打开灯、坐下、打电话、哭、然后笑」
- ✅「她推门进来,在门口停住,目光扫过空房间」

---

## 法则 3:运动矢量语言(具体可执行)

把「动起来」翻译成方向 + 速度 + 主体。

- 镜头:推(dolly in)、拉、左摇(pan left)、跟(tracking)、慢推(slow push)、甩(whip)
- 主体:「舞者旋转」「烟缓缓上升」「车从右驶入」
- 速度:缓 / 匀 / 急;给节奏感而非笼统「动感十足」

避免冲突运镜(同一句里既推又拉又摇)——会打架。

---

## 法则 4:主动写声音(Seedance 2.0 原生音频)

它一次前向就出双声道音频。**默认就该利用**,别只写画面。

- 写关键音效:脚步、关门、风、玻璃碎(给材质 + 时机)。
- 写对白/口播:直接写台词,模型会出口型(8+ 语言)。声音风格可配音频参考。
- 写 BGM 走向:情绪/速度,而非具体曲名。
- 让声音强化关键节拍:hook 那一秒配一个声音冲击点。

口播/对白片细节见 `storyboard-voice-control`;声音分层设计见 `storyboard-audio`。

---

## 法则 5:默认只写正向提示词(不用反向提示词)

**默认不写负向/反向提示词,把所有「不要 X」改写成对应的「要 Y」**,用**真实的技法词语**填充。多数视频模型对「不要 X」响应弱,反而会被你写出的词带跑;正向具体化才稳。先去搜真实电影/摄影技法文档(见下「先查技法文档」),用查到的技法词写正向目标。

- 「不要模糊」→「三脚架固定、水平稳定、对焦清晰锐利」
- 「不要杂乱背景」→「背景干净、浅景深虚化 f/2.8」
- 「不要多余的人」→「空旷无人的场景」
- 「不要太暗」→「柔光补光、曝光正常、暗部保留肩部质感」
- 「不要平」→「前中后三层纵深、空气透视、广角 24mm 强透视」

**先查技法文档:** 不管真人还是动画,先主动检索真实的电影/摄影技法资料(StudioBinder / American Cinematographer(ASC)/ No Film School / ShotDeck / Wikipedia「Cinematic techniques」;动画作画走 `animation-craft` 的权威源),用**实在的技法词**(焦段 mm、光圈 f值、景别、运镜 dolly/crane/steadicam/handheld、三点布光/伦勃朗/逆光、色温 K、调色 LUT)填正向提示词。

---

## 法则 6:控制提示词长度

太长的提示词会让模型失真。控制在**核心信息**:主导动作 + 关键镜头 + 关键声音 + 风格锚点(若首帧没覆盖)。其余交给参考图/视频/音频。

---

## 提示词骨架(可套用)

```
[主体 + 主导动作] + [镜头运动] + [关键声音/对白] + [节奏/情绪] +(仅文生视频时)[风格锚点]
```

示例(图生视频,首帧已定风格):
> 她抬眼直视镜头,镜头缓慢推近;窗外雨声渐起;气氛安静、克制。

示例(文生视频,需补风格):
> 霓虹街头,机车从右疾驰驶入,镜头跟拍;引擎轰鸣 + 电子鼓点;赛博朋克、湿地面反光、夜。
