# 门控规则与智能路由

## 为什么要门(Gate)

每道门 = 产出阶段成果 → 给用户看 → 等确认/修改 → 再前进。原因:

- **一致性向下游复利。** G3 角色锚点错了,G4 出图全错,G5 视频全废,G7 拼出来不能用。越往后,返工成本越高。
- **早抓问题最便宜。** 在剧本阶段改一句话,比在成片阶段重生 6 镜便宜 100 倍。
- **用户掌舵。** 创作决策(类型、画风、节奏、角色长相)是用户的,不是 agent 替他定。

**铁律:绝不因为用户"很急"就跳门。** 急 → 可以把门做"轻"(一句话确认),但不能没有。

## 每道门怎么过

1. 产出该阶段成果(剧本/镜头表/锚点/关键帧/片段/声音/粗剪/制片包)。
2. 用一句话总结这阶段做了什么 + 下阶段要做什么。
3. 明确问:「批准进入下一阶段,还是要改?」
4. 用户要改 → 回到本阶段重跑,**不丢下游已完成的工作**。
5. 批准 → 进下一阶段,并把本阶段关键产出(尤其锚点)带下去。

## 智能路由检测

开工先判断用户给了什么,跳到对应阶段,别从头问一遍:

| 信号(用户消息里出现) | 含义 | 跳到 |
|---|---|---|
| 只有一句创意/主题/情绪 | 概念 | G0 |
| 一段故事/小说/剧本文本 | 已有故事 | G2(拆分镜) |
| 「镜头表/分镜/shot list」+ 景别运镜 | 已有分镜 | G3 或 G4 |
| 角色立绘/设定 + 镜头表 | 已有设定 | G4(出图) |
| 一批关键帧图片 | 已有首帧 | G5(图生视频) |
| 一批视频片段 | 已有素材 | G6/G7(声音+拼接) |
| 「帮我把这些拼成片」 | 只缺成片 | G7 |

**判断细则:**
- 用户给的角色描述带**锁定属性**(具体脸/发/服装/道具)→ 跳过 G3。
- 用户给的镜头表带**运镜方向**(推/拉/跟 + 时长)→ 可跳到 G4。
- 拿不准 → 从 G0 开始,但用一两句快速确认带过前面阶段。

## 动画 vs 实拍的分叉

- **动画/卡通/赛璐珞片:** G4 叠 `director-anime-quality-boost`(画风),G5 叠 `animation-craft`(运动法则/计时节拍)。
- **实拍质感片:** G4 重 `director-lighting-continuity` / `storyboard-light-reconstruction`,G5 不叠 animation-craft,重 `storyboard-live-character-realism`(活人感)。

在 G0 就问清画风,决定后面走哪条分叉。
