# AI映像 制作ノート

アニメ工芸をモデルで使える映像作業・画像から動画(i2v)prompt・リテイクメモ・納品チェックに落とすとき読み込む。

## モデル受け渡し

映像 prompt は**尺の付いた視覚イベント**として書く:
- **最初の画:** 被写体・場所・スケール・ポーズ・カメラ距離・光源。
- **運動の経路:** 被写体/カメラがどこから始まり、どこへ、どれだけ速く動くか。
- **物理の原因:** 何が押す・引く・中断する・引き寄せる・阻む・変えるか。
- **反応:** 身体・小道具・環境・光・音・カメラの応答。
- **終わりの状態:** 最後の画が可視で、次カットに継承される状態。

「スタイル語付きの静止画一枚」を書かない。映像モデルには可視の変化が要る。

## prompt の尺(タイミング雛形)

実用の尺テンプレート:

| 尺 | 雛形 |
|---|---|
| 5s | 0–1s セットアップ、1–3s 本番、3–5s 反応/収まり |
| 8s | 0–2s セットアップ、2–5s 増幅、5–8s 転換/収まり |
| 10s | 0–2s セットアップ、2–6s 主運動、6–8s 衝突、8–10s 反応 |
| 15s | 0–3s セットアップ、3–7s 試み、7–11s 複雑化/転換、11–15s 帰結 |

全カットで全ビートを使い切る必要はない。可読性に必要な構造だけ使う。

## カメラメモ

カメラワークはカットを説明するもの:
- **フィックス(止め):** 芝居・構図・変形を明快に読ませる。
- **ゆっくり寄り(TU):** 圧力・気づき・親密・提示。
- **引き:** 帰結・孤独・スケール・余韻。
- **フォロー:** 追跡・通り抜け・巡回・隊列。
- **PAN / ティルト:** 関係の提示・視線追従・隠れた物の開示。
- **手持ちのドリフト:** 緊張・脆さ・主観的な不安定。

カメラ速度と被写体との関係を明記する。「躍動感あるカメラ」のような経路の無い曖昧語を使わない。

## 連続性チェック

マルチカットを生成する前にロックする:
- **キャラ同一性:** シルエット・顔/髪・服装・小道具・ポーズの癖。
- **場面同一性:** 固定の建築・光源・天候・地面/床の痕跡。
- **小道具状態:** 所有・位置・損傷・発光・液体・塵・布・開閉状態。
- **画面の向き:** 誰が左/右を向くか、誰が出入りするか、カメラ軸はどこか。
- **身体状態:** 疲労・傷・濡れ・塵・変形・情感の残留。
- **終わりの継承:** 次カットの最初の2秒が何を見せるべきか。

## 検証

生成クリップは最低2回観る:
- 通常速度で観客の可読性を見る。
- 最初の画・ピークの芝居・衝突・最後の画で一時停止する。

確認:
- カットに本当の運動があり、ドリフトする静止画でない。
- 主芝居が prompt 無しで読める。
- キャラと場面の同一性がドリフトしていない。
- カメラが要の芝居を隠していない。
- 光の方向・天候・小道具状態が一貫している。
- 最後の画が次カットに繋がる。
- 不要な文字・字幕・ロゴ・UI画面・透かしが出ていない。

## リテイク文言

リテイク prompt は「失敗」と「代替の振る舞い」を名指しする:

```text
問題:キャラの顔が振り向きで崩れる。
修正:全編で同じ卵形の輪郭・高い髪頂・黒い結び髪・左頬のほくろを保つ。視線だけを扉から落下する提灯へ移す。
```

```text
問題:カットが静止画のよう。
修正:0–8s に可視の進行を入れる——袖が風で舞い上がる、提灯が左から右へ揺れる、塵が床の隙間から立ち上る、カメラが中景から寄りへゆっくり TU。
```
