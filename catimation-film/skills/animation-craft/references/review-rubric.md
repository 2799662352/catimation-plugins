# アニメ映像レビュー rubric

生成されたアニメ映像・アニマティック・絵コンテ・AI動画 prompt の結果をレビューするとき読み込む。

## レビュー出力

まず所見を報告する。各問題に含めるもの:
- **重大度:** P0 / P1 / P2 / P3。
- **所見:** 画面上のどこが間違っているか。
- **リスク:** なぜこのカットを損なうか。
- **提案:** 最小で有用な修正。
- **検証:** 直ったことをどう確認するか。

所見が無いなら、そう明言し、未検証の領域を挙げる。

## 重大度

| 重大度 | 意味 |
|---|---|
| P0 | カットが使えない:被写体違い・読める芝居なし・深刻な同一性ドリフト・連続性崩壊・不安全/不要なコンテンツ。 |
| P1 | 重大な物語/ステージング/タイミング/連続性の失敗。納品を阻む。 |
| P2 | 明白な品質問題:タイミングが粗い・芝居が弱い・フリッカー・変形・カメラ不明瞭・終わりの状態が弱い。 |
| P3 | 物語リスクの低い磨きの提案。 |

## 何を確認するか

**物語の明快さ:**
- 観客は prompt を読まずに「何が変わったか」を言えるか。
- 最初の画は読めるか。
- 最後の画は新状態・反応・問いを落とすか。

**ステージング:**
- 明快な主芝居が一つあるか。
- シルエット・コントラスト・カメラ距離・光が可読性を支えるか。
- 小道具と障害の位置が因果を見せるか。

**タイミング:**
- 芝居の前に十分なセットアップがあるか。
- 衝突に明快な接触/変化のコマがあるか。
- 反応や収まりが読めるだけの尺があるか。
- 切りやビート変化が情報密度に対して速すぎないか。
- 重要な転換に**間(ま)**があるか。タメ・ツメが効いているか。

**芝居(演技):**
- ポーズ・視線・呼吸・手の動きが意図を表すか。
- 顔は身体を支えているか(身体の芝居の代用になっていないか)。
- 副次運動・フォロースルーが重さと情感を助けるか。

**連続性:**
- キャラ同一性は安定か。
- 場面レイアウト・光源・天候・画面の向きは追えるか。
- 小道具の所有・損傷・残留・身体状態がカットをまたいで継承されるか。

**制作の瑕疵:**
- フリッカー・溶け・変形・歪んだ手・崩れた顔・テクスチャの跳び。
- カメラが要の芝居を隠す。
- 不要な文字・ロゴ・字幕・アプリ画面・UIオーバーレイ・透かし。
- カット冒頭が空白、または末尾に継承できる状態が無い。

## 頻出所見

- クリップがカメラドリフト付きの静止画のよう → 尺付きの被写体/環境変化を足す。
- 主芝居がセットアップ理解の前に起きる → 読める最初の画を足す。
- カメラ中に顔が崩れる → 鎖骨/顔/髪のアンカーをロックし運動を簡素化。
- 場面がカット間でリセット → 塵・壊れた小道具・扉の状態・光方向・身体位置を継承。
- カットにエネルギーはあるが情報を伝えない → ビートに応じフィックス/寄り/フォロー/提示を選ぶ。
- リズムに音の手がかりが無い → 沈黙・衝突・尾音・環境ベッドの時序を足す。
- 動きが均等割りで機械的 → タメ・ツメと緩急を入れる。

## 検証チェックリスト

- 通常速度で一度観る。
- 最初の画・ピークの芝居・衝突のコマ・最後の画で一時停止。
- 最後の画と次カットの最初の画を比較。
- キャラ参照と照合して同一性アンカーを確認。
- 場面参照と照合して場面アンカーを確認。
- 不要な文字や UI 瑕疵を確認。
- リテイク prompt が「良くして」ではなく具体的な可視の代替を名指ししていることを確認。
